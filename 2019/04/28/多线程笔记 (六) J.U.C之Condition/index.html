  • Condition

JUC提供的Condition是为了更加灵活的对wait、notify操作。使用Condition可以在未达到某个条件时阻塞线程，条件满足时又可以唤醒线程。

  • Condition常用方法

await()：使当前线程在被唤醒或中断之前一直处于等待状态，同时会加入到Condition等待队列同时释放锁。

signal()：唤醒一个在等待队列中等待最长时间的节点（条件队列里的首节点）。该线程从等待方法返回前必须获得与Condition相关的锁。
